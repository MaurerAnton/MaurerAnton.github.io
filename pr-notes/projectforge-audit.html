1. jsonpath high + underscore high

jsonpath@1.3.0 тянет underscore@1.13.6, который содержит CWE-674/CWE-770 — неограниченная рекурсия в _.flatten и _.isEqual, вектор DoS (CVSS 5.9).

Где объявлено

ФайлСтрокаСодержимое
package.json:36 dependencies "jsonpath": "^1.3.0"
package-lock.json lock node_modules/jsonpath: 1.3.0 → underscore: 1.13.6

Где используется в исходниках

Нигде. Поиск по всем .js/.jsx/.ts/.tsx/.java/.kt внутри projectforge-webapp/src/ и src/ не нашёл ни одного import/require на jsonpath. Это мёртвая зависимость — не используется ни frontend, ни backend кодом.

Цепочка

package.json (dependencies)
  └─ jsonpath@1.3.0       ← мёртвая, нигде не импортируется
       └─ underscore@1.13.6  ← CWE-674/770, CVSS 5.9

Исправление

  1. Удалить "jsonpath": "^1.3.0" из dependencies в package.json:36
  2. Выполнить npm install — underscore уйдёт автоматически
Убирает 2 high уязвимости одним удалением.

3. ws moderate

ws@8.20.0 содержит CWE-908 — утечка неинициализированной памяти через неверную обработку заголовков (CVSS 4.4). Исправлено в ws@8.20.1.

Где объявлено

package-lock.json lock node_modules/jsdom: 24.1.3 → ws: ^8.18.0 → ws@8.20.0

Где используется в исходниках

ws — чисто транзитивная зависимость. Не импортируется напрямую. Тянется тестовой инфраструктурой:

vite.config.ts:14-17 test.environment: 'jsdom' — Vitest использует jsdom для DOM-окружения в тестах

package.json:91-93 Скрипты "test" и "test:watch" используют vitest с jsdom

Цепочка

package.json (devDependencies)
  └─ jsdom@^24.1.0 → resolves to 24.1.3
       └─ ws@^8.18.0 → resolves to 8.20.0  ← CWE-908 (fixed in 8.20.1)

Исправление

Два пути (выбрать один):

  1. overrides (рекомендуется): добавить в package.json: 
    {
  "overrides": {
    "ws": "8.20.1"
  }
}
  2. Обновить jsdom: "jsdom": "^26.0.0" — но это тянет breaking changes, требует отдельных тестов

4-5. cookie low + react-cookies low

cookie <0.7.0 содержит CWE-74 — принимает cookie-имя, путь и домен с невалидными символами, что открывает вектор для инъекций. Тянется через react-cookies@0.1.1.

Где объявлено

ФайлСтрокаСодержимое
package.json:48 dependencies (ошибка: должно быть devDeps) "react-cookies": "^0.1.1"
package.json:31 dependencies (ошибка: должно быть devDeps) "fetch-mock": "^12.6.0"
package.json:54 dependencies (ошибка: должно быть devDeps) "redux-mock-store": "^1.5.5"

Где используется в исходниках

Все три пакета используются исключительно в тестах, но находятся в секции dependencies (production) вместо devDependencies.

ФайлСтрокаСодержимое
authentication.test.js:2 import import fetchMock from 'fetch-mock';
authentication.test.js:3 import import cookies from 'react-cookies';

fetch-mock используется только в authentication.test.js. react-cookies — там же. redux-mock-store — там же и в reducer-тесте. Ни один из трёх пакетов не используется в production-коде.

Цепочка

package.json (dependencies)              ← должно быть devDependencies
  ├─ react-cookies@0.1.1
  │    └─ cookie@<0.7.0                 ← CWE-74
  ├─ fetch-mock@12.6.0                  ← тестовая утилита
  └─ redux-mock-store@1.5.5             ← тестовая утилита

Исправление

  1. Перенести react-cookies, fetch-mock, redux-mock-store из dependencies в devDependencies
  2. Добавить overrides в package.json: 
    {
  "overrides": {
    "cookie": "0.7.0"
  }
}
  3. Переписать authentication.test.js c fetch-mock/react-cookies на vi.fn() + global.fetch — тогда эти пакеты можно удалить полностью.
